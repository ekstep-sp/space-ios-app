The Java SE 9 divides the platform into a set of modules known as platform modules. These modules are further divided into following two types :

  • Standard modules

    • These are supported and intended to be used by developers

    • The names of the standard Java SE modules start with java.*

    • For example, the module named java.rmi defines the Remote Method Invocation API, and the module named java.logging defines the Java Logging API

    • It consists of standard API packages as well as non-standard API packages and also depends upon one or more non-standard modules

  • Non standard modules

    • The non-standard modules are specific to JDK

    • Their names start with jdk.*

    • Non-standard modules contain packages and specific JDK code, which may be distinct between various implementations of the Java Development Kit

You can list platform modules using the java --list-modules command, and look at an individual module with java --describe-module <module name> command.

 
JDK 9 also provides the following new tools:

  • jdeps - used to identify JAR and module dependencies

  • jlink - used to create custom runtime by combining application and JDK modules

  • jmod - used to create JMOD files which is a new format for packaging modules

These tools are present in the bin folder of JDK 9. We will discuss them later in the course.
